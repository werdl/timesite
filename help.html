how it works

the time is fetched from either an api, or a local Date() object. it utilises round trip time calculation, and makes exactly one request every second to the api. if it becomes too slow, it swaps to a local Date() object. every two seconds, if there has been a problem with the api, it tries again. if it succeeds, it swaps back to the api. if it fails, it continues to use the local Date() object.
